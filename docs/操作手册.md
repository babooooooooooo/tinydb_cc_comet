# tinydb 操作手册

> 本手册面向 tinydb v0.1.1 用户（开发者、嵌入式集成者、SQL 学习者），覆盖安装、SQL 方言、ACID 使用方式、REPL 操作、错误诊断与故障排查。

## 1. 安装

### 1.1 用户安装（仅运行时）

tinydb 运行时**零依赖**——完全使用 Python 3.11+ 标准库。从源码或 git 安装：

```bash
# 从 PyPI（一旦发布）
pip install tinydb

# 从 GitHub
pip install git+https://github.com/babooooooooooo/tinydb_cc_comet.git
```

安装后 Python 中可直接 `import tinydb`，命令行新增 `tinydb-repl` 命令。

### 1.2 开发者安装（带测试工具链）

```bash
git clone https://github.com/babooooooooooo/tinydb_cc_comet.git
cd tinydb_cc_comet
pip install -e ".[dev]"
```

`[dev]` 额外依赖：`pytest>=7`, `hypothesis>=6`, `pytest-cov>=4`。

### 1.3 系统要求

- Python ≥ 3.11
- 可选: readline（Unix 平台默认提供；用于 REPL 历史持久化）
- 操作系统: Linux / macOS / Windows（Windows REPL 无历史持久化）

## 2. 5 分钟快速上手

### 2.1 Python API

```python
import tinydb

# 内存数据库（重启即丢失）
with tinydb.Database(":memory:") as db:
    db.execute("CREATE TABLE users(id INT PRIMARY KEY, name TEXT, age INT)")
    db.execute("INSERT INTO users(id, name, age) VALUES (1, 'alice', 30)")
    db.execute("INSERT INTO users(id, name, age) VALUES (2, 'bob', 25)")
    for row in db.execute("SELECT * FROM users WHERE age >= 26"):
        print(row.id, row.name, row.age)

# 文件数据库（持久化）
with tinydb.Database("data.db") as db:
    db.execute("CREATE TABLE kv(k TEXT PRIMARY KEY, v INT)")
    db.execute("INSERT INTO kv(k, v) VALUES ('counter', 0)")
    for row in db.execute("SELECT * FROM kv"):
        print(row.k, row.v)
```

### 2.2 REPL

```bash
tinydb-repl                    # 内存数据库
tinydb-repl --database data.db # 文件数据库
```

交互样例：

```text
tinydb> CREATE TABLE accounts(id INT PRIMARY KEY, balance INT);
OK
tinydb> INSERT INTO accounts(id, balance) VALUES (1, 100);
OK
tinydb> INSERT INTO accounts(id, balance) VALUES (2, 50);
OK
tinydb> SELECT * FROM accounts WHERE balance >= 60;
id | balance
---+--------
 1 |    100
tinydb> .tables
accounts
tinydb> .schema accounts
CREATE TABLE accounts(id INT PRIMARY KEY, balance INT);
tinydb> .exit
```

## 3. SQL 方言速查

### 3.1 DDL

| 语句 | 形式 | 说明 |
|---|---|---|
| `CREATE TABLE` | `CREATE TABLE name(col1 TYPE, col2 TYPE(...), ...)` | 至少一列；列约束 `PRIMARY KEY` / `UNIQUE` / `NOT NULL` 可附加在列定义末尾 |
| `DROP TABLE` | `DROP TABLE name` | 删除表及其占用的页（释放空间） |

支持的 TYPE 见 [§ 4 类型系统](#4-类型系统)。

### 3.2 DML

| 语句 | 形式 | 说明 |
|---|---|---|
| `INSERT` | `INSERT INTO t(col1, col2, ...) VALUES (v1, v2, ...)` | 必须显式列出列；多行用逗号拼接 |
| `SELECT` | `SELECT [* \| col1, col2, ...] FROM t [WHERE cond] [ORDER BY col [ASC\|DESC]]` | 返回的每一行是不可变 `Row` 对象 |
| `UPDATE` | `UPDATE t SET col=val [, col=val ...] WHERE cond` | 受 WHERE 限制；无 WHERE 当前 MVP 不支持 |
| `DELETE` | `DELETE FROM t WHERE cond` | 必须有 WHERE |

### 3.3 WHERE 条件

tinydb 支持复合条件：

```sql
SELECT * FROM t
WHERE a = 1 AND b = 'x' OR NOT (c = 'y');
```

运算符：`=`, `<>`, `<`, `<=`, `>`, `>=`，用 `AND` / `OR` / `NOT` / 括号组合。

**重要**: `WHERE col = literal` 要求**精确同类型**——`INT ≠ SMALLINT ≠ BIGINT`，`FLOAT ≠ DOUBLE`，跨类型比较抛 `CodecError`（见 [§ 5 错误诊断](#5-错误诊断)）。

### 3.4 事务控制

每个 `db.execute()` 默认自动提交。如果需要多个语句作为一个原子单元：

```python
with tinydb.Database("data.db") as db:
    db.execute("BEGIN")
    try:
        db.execute("INSERT INTO accounts(id, balance) VALUES (2, 50)")
        db.execute("INSERT INTO accounts(id, balance) VALUES (3, 75)")
        db.execute("COMMIT")  # 一起生效
    except Exception:
        db.execute("ROLLBACK")
        raise
```

显式 `BEGIN` 后所有执行都加入当前事务，直到 `COMMIT` 或 `ROLLBACK`。如果中途抛异常，未显式 `COMMIT` 的更改也会自动回滚（autocommit 行为反过来——`BEGIN` 后的语句是手动提交）。

## 4. 类型系统

15 个列类型，全部走 [codec registry](#codec-contract)，任何边界越界都抛 `CodecError`。

| SQL 类型 | 参数 | 字节布局 |
|---|---|---|
| `INT` / `INTEGER` | — | 4 字节有符号 |
| `SMALLINT` | — | 2 字节有符号（±32767） |
| `BIGINT` | — | 8 字节有符号 |
| `FLOAT` / `REAL` | — | 4 字节 IEEE 754 单精度（≈7 位精度） |
| `DOUBLE` | — | 8 字节 IEEE 754 双精度 |
| `TEXT` | — | 2 字节长度前缀 + UTF-8 |
| `VARCHAR(N)` | N ≥ 1 | 2 字节长度前缀 + UTF-8，最长 N 字节 |
| `CHAR(N)` | N ≥ 1 | 2 字节长度前缀 + UTF-8，写入右补空格至 N 字节（SQL92 PAD SPACE） |
| `BOOL` / `BOOLEAN` | — | 1 字节 |
| `DECIMAL(p, s)` | 1 ≤ p ≤ 18, 0 ≤ s < p | 8 字节 scaled int64（最高 18 位精度） |
| `DATE` | — | 4 字节 days since 1970-01-01 UTC |
| `TIME` | — | 4 字节 seconds since midnight UTC |
| `TIMESTAMP` | — | 8 字节 seconds since 1970-01-01 UTC |

**类型化字面量前缀**：

```sql
INSERT INTO events(d, t, ts, amt) VALUES (
  DATE '2026-07-16',
  TIME '14:30:00',
  TIMESTAMP '2026-07-16 14:30:00',
  DECIMAL '99.99'
);
```

## 5. 错误诊断

### 5.1 异常层次

```
TinydbError                   顶层
├── ParseError                解析阶段 (line/col 信息)
├── TokenError                词法阶段 (line/col 信息)
├── ExecutionError
│   └── ConstraintViolation   NOT NULL / UNIQUE / PK 违规
├── InvalidDatabaseFile       DB 文件结构损坏或字段格式不可识别
├── UnsupportedSchemaVersion  DB 版本过新，旧代码不认识
├── SchemaMismatch            DB 版本过旧，需要迁移
├── PageFull                  页容量耗尽
├── CatalogFull               catalog 页溢出 (> 4 KB)
└── CodecError (来自 tinydb.type_system) — type-level validation failure
        注：CodecError 多继承 TypeError/ValueError/OverflowError
```

所有 `TinydbError` 子类共享一个根异常，便于统一捕获。

### 5.2 Codec 异常合约（关键）

codec 校验失败时**只**抛 `tinydb.type_system.CodecError`。从 v0.1.1 起，**每个** codec 的 `encode_py` 都把 `self.validate(value)` 作为第一行执行——错误类型错误、值越界、长度溢出、精度溢出、`FLOAT`/`DOUBLE` 收到 `NaN`/`Infinity`，所有 type-level 失败都收敛到 `CodecError`。任何直接 `codec.encode_py(...)` 调用都不再泄漏 `AttributeError`/`TypeError`/`OverflowError`/`ValueError`。

| 触发条件 | 抛 `CodecError` 示例 |
|---|---|
| `INT` / `SMALLINT` / `BIGINT` 越界 | `CodecError: INT out of range: 2147483648` |
| `INT` 收到 `bool` (`True`/`False`) | `CodecError: expected int for INT, got bool` |
| `FLOAT` / `DOUBLE` 收到 `NaN` / `Infinity` | `CodecError: FLOAT inf/NaN not allowed: nan` |
| `VARCHAR(N)` 字符串长度 > N（按 UTF-8 字节） | `CodecError: VARCHAR(10) length 11 exceeds max` |
| `CHAR(N)` 同上（v0.1.1 之前会误报为 `VARCHAR(N)`） | `CodecError: CHAR(5) length 6 exceeds max` |
| `DECIMAL(p, s)` 数值超精度（v0.1.1 之前抛裸 `OverflowError`） | `CodecError: DECIMAL(10,2) value 1000.00 out of range` |
| `DECIMAL(p, s)` scaled 内部溢出 | `CodecError: DECIMAL(18,6) scaled value overflow` |
| `TIME` 越界（v0.1.1 之前抛 `ValueError`） | `CodecError: TIME out of range: 86400` |
| `WHERE col = literal` 跨类型 | `CodecError: type mismatch: ...` |
| 索引路径下 `WHERE id = 2147483648` on `INT PRIMARY KEY`（v0.1.1 之前静默 `[]`） | `CodecError: INT out of range: 2147483648` |

`CodecError` 是 type-level 错误的**唯一**异常源。**新代码应捕获 `CodecError`**，不要捕获 `TypeError`/`OverflowError`/`ValueError`。

> **向后兼容**：`CodecError` 多继承 `(TypeError, ValueError, OverflowError)`——已有 `except (TypeError, ValueError, OverflowError)` 的旧代码会自动捕获 `CodecError`。**注意**：v0.1.1 之前多继承是 load-bearing 的（部分 codec 抛父类），v0.1.1 之后所有 codec 都直接抛 `CodecError`，多继承仅作 backward-compat shim。

### 5.3 常见异常 → 处理建议

| 异常 | 含义 | 建议处理 |
|---|---|---|
| `ParseError` / `TokenError` | SQL 语法或词法错误 | 修正 SQL；`.line` / `.col` 字段定位出错位置 |
| `ConstraintViolation(kind='null')` | 写入 NULL 到非空列 | 检查 INSERT/UPDATE 是否提供了该列；或允许该列 NULL |
| `ConstraintViolation(kind='unique')` | 违反 UNIQUE 约束 | 查重；如合理则 `UPDATE` 而非 `INSERT` |
| `ConstraintViolation(kind='duplicate_pk')` | PK 重复 | 用 `WHERE id = ...` 先查；决定 `UPDATE` 还是抛错 |
| `InvalidDatabaseFile` | 文件已损坏或字段格式不可识别（含 schema v1 旧 `[name, type]` 数组形态） | 若是 schema 升级：从 v1 文件迁移到 `Column.to_dict()` 形态 |
| `SchemaMismatch` | DB 文件版本高于当前代码支持的最高版本 | 升级 tinydb |
| `UnsupportedSchemaVersion` | 同上 | 同上 |
| `PageFull` | 单页耗尽 | 减小单行大小；`tinydb-engine-v2` 引入 overflow chain 自动扩展 |
| `CatalogFull` | catalog 页超过 4 KB | 删除冗余表；或将 catalog 多页化（roadmap） |
| `CodecError` | type-level 校验失败 | 见 [§ 5.2](#52-codec-异常合约关键) |

### 5.4 异常处理范式

```python
import tinydb

try:
    db.execute("INSERT INTO t(i) VALUES (2**40)")
except tinydb.errors.CodecError as e:
    print(f"type-level error: {e}")
except tinydb.errors.ConstraintViolation as e:
    print(f"constraint failed: kind={e.kind}, column={e.column}")
except tinydb.errors.TinydbError as e:
    print(f"other tinydb error: {type(e).__name__}: {e}")
```

## 6. REPL 操作

### 6.1 启动

```bash
tinydb-repl                    # 内存数据库
tinydb-repl --database data.db # 打开/创建文件数据库
```

进入后看到 `tinydb>` 提示符。文件数据库的所有修改自动 `fsync`。

### 6.2 多行输入

未闭合的单引号 `'` 或未配对的 `(` 会进入续行模式 `...>`：

```text
tinydb> SELECT * FROM users
...> WHERE name = 'alice
...> ' OR age > 30;
```

只要 SQL 一致，多行粘贴是允许的（粘贴多行 SQL 即可）。

### 6.3 元命令

| 命令 | 说明 |
|---|---|
| `.help` | 列出所有元命令 |
| `.exit` / `.quit` | 退出 REPL |
| `.tables` | 列出所有表名（每行一个） |
| `.schema <name>` | 反向生成 `CREATE TABLE` 语句 |
| `.read <path>` | 读取并执行 UTF-8 SQL 脚本文件 |

### 6.4 错误显示

执行错误显示为单行 `ERROR: <ExceptionClass>: <message>`，**不退出** session——直接重新输入 SQL 即可。

```text
tinydb> INSERT INTO t(i) VALUES (2147483648);
ERROR: CodecError: INT out of range: 2147483648
tinydb> SELECT * FROM t;
[results...]
```

### 6.5 历史记录

- Unix: `~/.tinydb_history`（自动加载/保存）
- Windows / 无 readline: 内置 `input()` 兜底——SQL、元命令、输出全工作，但历史不持久化

## 7. 性能特性与限制

### 7.1 适合的场景

- 嵌入式数据库（教学 / 工具 / 单进程服务）
- 中小数据集（单库 < 1 GB）
- 需要 ACID 而不想拖入 SQLite（更轻、更可读、易于 fork/扩展）
- 学习目的——逐行可读的存储引擎实现

### 7.2 不适合

- 多进程/多线程并发（无隔离级别、无并发控制）
- 大数据量（无 page eviction，DB 常驻内存）
- 高 QPS 写入（autocommit 默认 `fsync`，受限于磁盘 `fsync` 速度）

详见 [`MVP_LIMITATIONS.md`](MVP_LIMITATIONS.md)。

## 8. 故障排查

### 8.1 `ModuleNotFoundError: No module named 'tinydb'`

未安装 tinydb。重新运行 `pip install -e .` 或在脚本前加入 `sys.path.append(...)`。

### 8.2 `tinydb.errors.CodecError: INT out of range: <value>`

写入 INT 超出 4 字节有符号范围（−2³¹ .. 2³¹−1）。换 `BIGINT`。

### 8.3 `tinydb.errors.CodecError: VARCHAR(N) length M exceeds max`

UTF-8 字节长度超限。注意中文字符是 3 字节/字符。`CHAR` 同样规则。

### 8.4 `tinydb.errors.ConstraintViolation: kind='duplicate_pk'`

主键重复。先 `SELECT` 查询是否已存在。如需要替换，先 `DELETE` 再 `INSERT`，或用 `UPDATE`。

### 8.5 `tinydb.errors.InvalidDatabaseFile: unrecognized column entry: ['id', 'INT'] ...`

文件是 schema v1 格式——遗留 `["id", "INT"]` 数组形态。当前 tinydb 只读 v2+ 的 `Column.to_dict()` 对象形态。先迁移：

```python
# 一次性迁移：将 v1 catalog 导出为 Python 数据，导入到 v2 新数据库
import json
from tinydb import Database
from tinydb.catalog import Column

with open("legacy_data.json") as f:
    legacy = json.load(f)

with Database("new.db") as db:
    for tname, info in legacy["tables"].items():
        cols = [Column(name=n, type=ty, primary_key=(n == 'id')) for n, ty in info["schema"]]
        db.execute(f"CREATE TABLE {tname}(...)")  # 显式列定义
        for row in info["rows"]:
            # 逐行 INSERT ...
            ...
```

### 8.6 `tinydb.errors.SchemaMismatch: ...`

DB 文件版本高于当前代码识别范围。升级 tinydb：

```bash
pip install -U tinydb
```

### 8.7 REPL 没有历史

平台无 `readline` 模块（Windows 默认）。换 Unix 平台或安装 `pyreadline` / `gnureadline`。

### 8.8 测试运行报错 `pytest: command not found`

未安装 dev 依赖：

```bash
pip install -e ".[dev]"
```
