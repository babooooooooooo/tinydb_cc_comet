## 6. 公共 API 影响

**无破坏性变更**。

| API surface | 行为变化 |
|-------------|---------|
| `db.execute(...)` | 所有 type-level 失败现在统一抛 `CodecError`（不再泄漏 `AttributeError`/`TypeError`/`OverflowError`） |
| 索引路径 `WHERE col = bad_literal` | 现在抛 `CodecError`（v0.1.1 之前静默 `[]`）—— 严格意义上是**行为修正**，与 `WHERE col = good_literal` 在非索引路径上的错误行为一致 |
| `Catalog.create_table` 错误消息 | `"create_table expects Column ..."` → `"TableInfo expects Column instances ..."`（错误源从 create_table 移到 `TableInfo.__post_init__`） |
| `codec.encode_py(value)` 直接调用 | 现在所有 codec 在 value 类型错误时抛 `CodecError` 而非 `AttributeError`/`TypeError`/`OverflowError` |
| `CodecError` 多继承 `(TypeError, ValueError, OverflowError)` | 保留为 backward-compat shim；v0.1.1 之后**不** load-bearing（codec 不再直接抛父类） |

`CodecError` 多继承 shim 在 v0.1.1 之后变成**纯 backward-compat**——未来可以安全移除，无需修改 codec 实现。

## 7. 已接受的非阻断 finding

- **V5** — INT/FLOAT `encode_py` 在 INSERT 热路径上有双 validate（executor 预校验 + encode_py 内层 `self.validate`）。属 deliberate safety net：给 REPL、未来 codepath、直接调用等非预校验 caller 一致保护。影响为 ~2x isinstance + range 检查 / INT cell / INSERT。**接受**。
